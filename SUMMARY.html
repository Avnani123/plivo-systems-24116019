System Architecture & Design Summary

System: Low-Latency UDP Audio Transport & Playout Engine

Core Strategy & Constraints

  • Bandwidth Overhead Target: ≤ 2.0× (Achieved: 1.54×)
  • Deadline Miss Cap: ≤ 1.00%
  • Grading Delay Recommendation: delay_ms = 50

1. High-Level Architecture

The system comprises a dual-process C application operating over UDP loopback sockets:

Harness Source (47010) → SENDER (47001) → Relay (47002) → RECEIVER (47020) → Harness Player

2. System-Level Design Choices

A. Forward Error Correction (FEC)

Rather than relying on retransmissions (ARQ), which add at least one full Round-Trip Time (RTT) across a hostile network, we employ 1-D XOR Forward Error Correction:

  • For every pair of media frames $(2k, 2k+1)$, an XOR parity packet is generated: Parity = Payload[2k] ^ Payload[2k+1].
  • Parity packets are flagged with a high bit (0x80000000) in the sequence header.
  • If Frame $2k$ is dropped by the network, it is mathematically reconstructed the moment Frame $2k+1$ and the Parity packet land at the receiver.

B. Playout Timing & Synchronization

  • MONOTONIC Clock Anchoring: All timing logic uses CLOCK_MONOTONIC to guard against system clock adjustments (NTP shifts).
  • Estimated Start Anchor ($E_0$): The receiver calculates $E_0 = T_{recv} - (\text{seq} \times 20\text{ms})$ across initial incoming frames to accurately anchor the media start time.
  • High-Precision Playout Loop: Playout pacing is governed by clock_nanosleep(CLOCK_MONOTONIC, TIMER_ABSTIME, ...) to prevent sleep drift and thread scheduling variance.

C. Memory Ownership & Thread Safety

  • Locking Paradigm: Non-blocking network I/O feeds a circular jitter buffer protected by POSIX mutexes (pthread_mutex_t).
  • Zero-Allocation Playout: Buffering uses fixed-size ring structures allocated statically at startup to avoid runtime dynamic heap allocation delays (malloc free execution paths).
